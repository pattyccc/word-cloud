作業二：文字雲
一、 設計邏輯與目標
本程式旨在展示Hash資料結構在處理大量文本數據時的高效性能。透過建立一個圖形化介面（GUI），使用者可以貼上任意英文長篇報導（本範例使用日本漢堡王加盟擴張之新聞），程式會自動分析字頻，並產出一張視覺化文字雲。為了提升資訊的精確度，程式加入了 Stop Words（無意義字）過濾機制，確保如 "the"、"is"、"and" 等高頻但低資訊價值的字詞不會干擾分析結果。

二、 核心技術：Hash的應用
在文字頻率統計的過程中，Hash技術發揮了關鍵作用:高效搜尋與存取：相較於陣列需要循序搜尋，Hash表能透過Hash function計算，在O(1)的時間複雜度內直接定位單字存放的記憶體位址，大幅提升統計速度。
Key 的唯一性：程式將每個單字設為 Hash Key。利用雜湊鍵唯一的特性，當程式再次遇到相同單字時，會直接在原位址更新其出現次數，確保計數的準確性且不重複佔用記憶體。
過濾邏輯：在資料存入雜湊表前，程式會先與 Stop Words 集合進行雜湊查找，能瞬間判斷該字是否應被排除，確保最終文字雲的精煉度。

三、 成果展示與特點-圖形介面（GUI）：使用 tkinter 建立簡潔的視窗，並針對macOS系統貼上功能，提升操作體驗。 
視覺化效果：利用 wordcloud 套件，將雜湊表中的頻率轉化為字體大小。頻率愈高的字（如 "Japan"、"Burger"、"Restaurants"）在圖片中愈顯眼。 
輔助功能：生成視窗附帶工具列，支援圖片存檔、調整邊距及縮放檢視，方便使用者產出高品質的分析圖表。
